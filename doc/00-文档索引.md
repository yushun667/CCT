## 优先级定义

| 等级 | 说明 |
|------|------|
| **P0（必需）** | 产品核心功能，必须实现，影响产品是否可交付 |
| **P1（重要）** | 显著增强用户体验或效率，建议在首个正式版本中实现 |
| **P2（可选）** | 高级特性，可根据资源和时间情况在后续版本中迭代 |

## 需求编号体系

所有需求条目使用统一编号格式：

- **功能需求**：`FR<模块编号>.<序号>` — 如 `FR1.1`、`FR7.4`
- **非功能需求**：`NFR<类别编号>.<序号>` — 如 `NFR1.1`、`NFR4.3`
- **用例编号**：`UC<模块编号>.<序号>` — 如 `UC1.1`、`UC7.2`
