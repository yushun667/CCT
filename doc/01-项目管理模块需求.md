# 01 — 项目管理模块细化需求

> 对应需求规范：§3.1 项目管理（FR1.1 ~ FR1.8）  
> 优先级范围：P0 ~ P1

---

## 1. 模块概述

项目管理模块是用户使用平台的入口，负责本地/远程项目的创建、配置、持久化存储、SSH 远程连接管理和远程 Agent 生命周期管理。它为后续的源码解析、查询与可视化分析提供基础数据源。

### 1.1 职责边界

- 管理项目的生命周期（创建 → 配置 → 使用 → 归档/删除）
- 管理本地源码目录访问
- **管理 SSH 远程连接（连接、认证、会话保持）**
- **管理远程 Agent 的部署、启动、停止和升级**
- **通过 SSH/SFTP 浏览远程文件系统**
- 管理编译数据库的导入与验证（本地和远程）
- 管理认证信息的安全存储
- 提供项目列表与快速切换功能

### 1.2 与其他模块的关系

| 下游模块 | 依赖内容 |
|----------|----------|
| 源码解析与索引 | 项目源码路径（本地路径或远程路径 + SSH 连接）、编译数据库路径 |
| 查询与检索 | 项目上下文（当前活跃项目）、索引数据 |
| 可视化分析 | 项目范围、模块定义 |
| 代码编辑器 | 文件内容（本地读取或 SSH/SFTP 远程读取） |
| AI 智能助手 | 当前项目上下文 |

## 2. 用例详细描述

### UC1.1 创建本地项目

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **前置条件** | 用户已打开应用 |
| **触发方式** | 菜单「文件 → 新建项目 → 本地项目」或首页「新建项目」按钮 |
| **优先级** | P0 |

**主流程**：

1. 用户点击"新建本地项目"。
2. 系统弹出项目创建向导对话框。
3. 用户输入项目名称（必填，不超过 128 字符，不允许特殊字符 `/\:*?"<>|`）。
4. 用户选择本地源码根目录（通过系统文件选择器）。
5. 系统验证目录是否存在且可读。
6. 用户可选导入编译数据库（见 UC1.3）。
7. 用户点击"完成"，系统创建项目配置并保存。
8. 系统自动切换到新创建的项目，在主界面显示项目信息。

**异常流程**：

- **E1**：目录不存在或不可读 → 提示"所选目录无效，请重新选择"。
- **E2**：项目名称已存在 → 提示"项目名称已存在，请使用其他名称"。
- **E3**：目录为空或不含 C/C++ 源文件 → 警告"目录中未检测到 C/C++ 源文件，是否继续？"

**验收标准**：

- [ ] 能成功创建本地项目并保存配置
- [ ] 项目名称校验规则正确生效
- [ ] 目录不存在时给出明确错误提示
- [ ] 项目创建后自动切换为当前活跃项目

### UC1.2 创建远程项目（SSH 直连）

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色（尤其是远程团队协作成员） |
| **前置条件** | 用户已打开应用，且有可通过 SSH 登录的远程服务器 |
| **触发方式** | 菜单「文件 → 新建项目 → 远程项目」 |
| **优先级** | P0 |

**主流程**：

1. 用户点击"新建远程项目"。
2. 系统弹出远程项目创建向导。
3. 用户输入项目名称（规则同 UC1.1）。
4. 用户输入 SSH 连接信息：
   - **主机地址**（必填）：IP 或域名
   - **端口**（可选，默认 22）
   - **用户名**（必填）
   - **认证方式**：
     - **SSH 密钥**（推荐）：选择本地私钥文件路径 + 密码短语（可选）
     - **密码认证**：输入登录密码
     - **SSH Agent**：使用系统 SSH Agent 中的密钥
   - **跳板机配置**（可选）：ProxyJump 跳板机地址
5. 用户点击"测试连接"，系统通过 SSH 尝试连接远程服务器。
6. 连接成功后，系统显示远程服务器基本信息（操作系统、CPU 核心数、可用内存等）。
7. 用户通过远程文件浏览器选择远程服务器上的源码根目录。
8. 系统检测远程目录是否包含 C/C++ 源文件。
9. 用户可选指定远程编译数据库路径（见 UC1.3）。
10. 用户点击"完成"，系统创建项目配置，保存 SSH 连接信息（加密存储）。
11. 系统提示"是否立即部署远程解析 Agent？"（见 UC1.6）。

**异常流程**：

- **E1**：SSH 连接失败 → 提示具体错误：
  - "认证失败：用户名或密钥不正确"
  - "连接超时：无法连接到 <host>:<port>"
  - "Host key 验证失败：远程主机指纹不匹配"
- **E2**：远程目录不存在或无权限 → 提示"远程目录 <path> 不存在或当前用户无读取权限"。
- **E3**：远程服务器资源不足 → 警告"远程服务器可用内存不足 2GB，解析大型项目可能失败"。

**验收标准**：

- [ ] 支持 SSH 密钥认证、密码认证、SSH Agent 三种认证方式
- [ ] 支持跳板机（ProxyJump）连接
- [ ] SSH 连接信息加密存储（不出现在日志和配置明文中）
- [ ] 连接测试能正确反馈成功/失败及具体原因
- [ ] 能通过 SFTP 浏览远程文件目录
- [ ] Host Key 首次连接时提示用户确认指纹

### UC1.3 导入编译数据库

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **前置条件** | 项目已创建 |
| **触发方式** | 项目创建向导中的可选步骤 / 项目设置面板 |
| **优先级** | P0 |

**主流程（本地项目）**：

1. 用户在项目设置中选择"导入编译数据库"。
2. 系统提供两种方式：
   - **手动选择**：用户通过文件选择器选取 `compile_commands.json` 文件。
   - **自动检测**：系统扫描项目根目录及常见构建目录（`build/`、`out/`、`cmake-build-*/`），自动列出发现的编译数据库文件。
3. 后续步骤同原流程（校验 JSON → 统计 → 确认导入）。

**主流程（远程项目）**：

1. 用户在项目设置中选择"导入编译数据库"。
2. 系统通过 SSH/SFTP 提供远程文件浏览器：
   - **手动选择**：用户在远程文件浏览器中导航并选取 `compile_commands.json` 文件。
   - **自动检测**：系统通过 SSH 在远程服务器上扫描项目根目录及常见构建目录，列出发现的编译数据库文件。
   - **远程生成**：用户可通过 SSH 在远程服务器上执行编译数据库生成命令（如 `bear make`、`cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=ON ..`），系统提供命令输入框和实时输出显示。
3. 系统通过 SSH 读取文件内容进行校验。
4. 编译数据库路径记录为远程路径（供远程 Agent 使用）。

**验收标准**：

- [ ] 远程项目可通过 SFTP 浏览并选择编译数据库文件
- [ ] 支持在远程服务器上执行命令生成编译数据库
- [ ] 远程文件的 JSON 校验和摘要功能正常
- [ ] 编译数据库路径正确存储（区分本地路径和远程路径）

### UC1.4 项目配置管理

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **优先级** | P0 |

**配置项清单**：

| 配置项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `name` | String | 是 | 项目名称 |
| `type` | Enum | 是 | `local` / `remote` |
| `source_root` | Path | 是 | 源码根目录（本地路径或远程路径） |
| `ssh_config` | Object | 远程必填 | SSH 连接配置 |
| `compile_db_path` | Path | 否 | 编译数据库路径 |
| `module_definitions` | Array | 否 | 用户自定义模块定义 |
| `agent_config` | Object | 远程必填 | 远程 Agent 配置 |
| `created_at` | DateTime | 自动 | 创建时间 |
| `updated_at` | DateTime | 自动 | 最后更新时间 |

**持久化方案**：

- 项目配置以 JSON 文件存储在应用数据目录下。
- 路径：`<app_data>/projects/<project_id>/project.json`
- SSH 认证信息单独存储并加密：`<app_data>/projects/<project_id>/auth.enc`
- 加密方式：使用操作系统密钥链（macOS Keychain / Windows Credential Manager / Linux Secret Service）或 AES-256 加密。
- SSH Host Key 指纹存储在 `<app_data>/known_hosts`。

**验收标准**：

- [ ] 所有配置项可通过 UI 设置面板修改
- [ ] 配置修改后立即持久化
- [ ] 应用重启后配置完整恢复
- [ ] SSH 认证信息不以明文形式出现在任何日志或配置文件中

### UC1.5 多项目切换

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **优先级** | P1 |

**主流程**：

1. 用户点击左上角的项目选择器（下拉菜单）或侧边栏的项目列表。
2. 系统显示所有已创建的项目列表，包括项目名称、类型图标（本地/远程）、最后更新时间、连接状态（远程项目）。
3. 用户选择目标项目。
4. 系统切换当前上下文到目标项目：
   - 加载项目配置
   - 加载已有索引（如有）
   - 更新文件浏览器视图（远程项目需要建立 SSH 连接）
   - 更新图视图（清空或加载上次状态）
5. 对于远程项目，系统自动尝试建立 SSH 连接。
6. 如果目标项目尚未解析，系统提示"该项目尚未解析，是否立即开始？"

**验收标准**：

- [ ] 项目切换响应时间 < 2 秒（不含 SSH 连接和索引加载）
- [ ] 切换时保留原项目的视图状态（下次切回可恢复）
- [ ] 项目列表支持排序（名称/时间）和搜索过滤
- [ ] 远程项目显示连接状态（已连接/断开/连接中）

### UC1.6 远程 Agent 管理

| 项目 | 内容 |
|------|------|
| **参与者** | 远程团队协作成员、架构师 |
| **前置条件** | 远程项目已创建，SSH 连接正常 |
| **优先级** | P0 |

**Agent 概述**：

远程 Agent（`cct-agent`）是一个独立的 Rust 编译的命令行程序，部署在远程服务器上。它的职责：

- 接收来自桌面端的解析请求（通过 SSH 隧道上的 RPC）
- 在远程服务器上读取源码文件
- 使用 Clang LibTooling 执行源码解析
- 将解析结果（索引数据库）打包传输回桌面端
- 报告解析进度

**主流程 — 部署 Agent**：

1. 系统检测远程服务器的操作系统和架构（`uname -sm`）。
2. 系统从本地应用的内置资源中选取对应平台的 Agent 二进制文件。
3. 系统通过 SFTP 将 Agent 上传到远程服务器的指定目录（默认 `~/.cct/agent/`）。
4. 系统通过 SSH 设置可执行权限（`chmod +x`）。
5. 系统通过 SSH 启动 Agent 并验证版本兼容性。
6. 部署完成后显示状态："Agent 已就绪 (v1.0.0, Linux x86_64)"。

**主流程 — 启动/停止 Agent**：

- **启动**：通过 SSH 执行 Agent 命令，Agent 以后台守护进程方式运行或作为 SSH 命令的子进程运行。
- **停止**：通过 SSH 发送停止信号或关闭 SSH 会话。
- **状态检查**：通过 SSH 查询 Agent 进程是否运行中。

**主流程 — 升级 Agent**：

1. 桌面端检测到内置 Agent 版本高于远程 Agent 版本。
2. 提示用户"远程 Agent 版本 (v1.0.0) 低于当前版本 (v1.1.0)，是否升级？"
3. 用户确认后，系统上传新版本并替换旧版本。
4. 重启 Agent。

**Agent 运行模式**：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **按需启动** | 每次解析时通过 SSH 启动 Agent，解析完成后退出 | 偶尔使用，资源友好 |
| **常驻守护** | Agent 作为后台服务持续运行，监听来自桌面端的请求 | 频繁使用，响应快 |

**Agent 配置**：

```
AgentConfig {
  install_path: String            // Agent 安装目录，默认 ~/.cct/agent/
  mode: AgentMode                 // OnDemand | Daemon
  max_threads: Option<u32>        // 最大解析线程数（默认使用服务器核心数-1）
  max_memory_mb: Option<u64>      // 最大内存使用量（MB）
  log_level: String               // Agent 日志级别
  data_dir: String                // Agent 工作目录（存放临时数据），默认 ~/.cct/data/
}
```

**异常流程**：

- **E1**：远程服务器架构不受支持（如 MIPS） → 提示"不支持的远程服务器架构：<arch>"。
- **E2**：SFTP 上传失败（磁盘满/权限） → 提示具体原因。
- **E3**：Agent 启动失败 → 显示远程 stderr 输出，帮助排查。
- **E4**：Agent 版本不兼容 → 强制升级提示。

**验收标准**：

- [ ] 支持自动检测远程服务器的操作系统和架构
- [ ] 支持至少 Linux x86_64 和 Linux aarch64 两种平台的 Agent
- [ ] Agent 部署过程有进度反馈（上传进度、部署状态）
- [ ] Agent 版本不兼容时能正确提示升级
- [ ] Agent 崩溃后桌面端能检测到并提示重启

### UC1.7 远程文件浏览

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **前置条件** | 远程项目已创建，SSH 连接正常 |
| **优先级** | P0 |

**主流程**：

1. 用户打开远程项目后，左侧文件浏览器通过 SFTP 显示远程目录结构。
2. 目录树按需加载（展开目录时才读取子项，避免一次性加载大量文件列表）。
3. 用户单击文件可在编辑器中预览（通过 SFTP 读取文件内容）。
4. 文件浏览器显示的信息：文件名、大小、修改时间、类型图标。
5. 支持在文件浏览器中搜索文件名。

**性能优化**：

- SFTP 目录列表结果缓存（TTL 可配置，默认 60 秒）
- 文件内容读取使用懒加载，仅在用户打开时读取
- 大文件（> 10MB）提示"文件较大，是否继续加载？"
- SSH 连接复用（ControlMaster），避免频繁建连

**验收标准**：

- [ ] 远程目录树正确显示文件和子目录
- [ ] 目录展开响应时间 < 2 秒（正常网络条件下）
- [ ] 远程文件可在编辑器中正确打开和显示
- [ ] 支持文件名搜索

**验收标准**：

- [ ] SSH 连接状态实时更新

## 3. 数据模型

### 3.1 项目实体

```
Project {
  id: UUID                      // 唯一标识
  name: String                  // 项目名称
  project_type: ProjectType     // Local | Remote
  source_root: String           // 源码根目录（本地路径或远程绝对路径）
  ssh_config: Option<SSHConfig> // SSH 配置（仅远程项目）
  agent_config: Option<AgentConfig> // Agent 配置（仅远程项目）
  compile_db_path: Option<String>   // 编译数据库路径
  module_definitions: Vec<ModuleDef>   // 模块定义列表
  created_at: DateTime
  updated_at: DateTime
  last_parse_at: Option<DateTime>      // 最后解析时间
  parse_status: ParseStatus            // NotStarted | InProgress | Completed | Failed
}
```

### 3.2 SSH 配置

```
SSHConfig {
  host: String                  // 主机地址
  port: u16                     // 端口，默认 22
  username: String              // 用户名
  auth_method: SSHAuthMethod    // Key | Password | Agent
  key_path: Option<String>      // SSH 私钥文件路径（Key 认证）
  auth_ref: String              // 指向加密认证信息（密码/密码短语）的引用 ID
  proxy_jump: Option<String>    // 跳板机地址（如 user@bastion:22）
  keep_alive_interval: u32      // 心跳间隔（秒），默认 30
  connect_timeout: u32          // 连接超时（秒），默认 15
  known_hosts_policy: HostKeyPolicy  // Accept | Reject | AskUser
}

SSHAuthMethod {
  Key { key_path: String, passphrase_ref: Option<String> }
  Password { password_ref: String }
  Agent  // 使用系统 SSH Agent
}
```

### 3.3 Agent 配置

```
AgentConfig {
  install_path: String            // 远程安装路径，默认 ~/.cct/agent/
  data_dir: String                // 远程工作目录，默认 ~/.cct/data/
  mode: AgentMode                 // OnDemand | Daemon
  max_threads: Option<u32>        // 最大线程数
  max_memory_mb: Option<u64>      // 最大内存
  version: Option<String>         // 已安装版本
}
```

### 3.4 模块定义

```
ModuleDef {
  name: String                  // 模块名称
  match_type: MatchType         // Directory | FileList | Regex
  patterns: Vec<String>         // 匹配模式列表
  color: Option<String>         // 可视化时的显示颜色
  description: Option<String>   // 模块描述
}
```

## 4. 接口设计（前后端通信）

项目管理模块通过 Tauri Command 暴露以下接口：

| 命令名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| `create_local_project` | `name, source_root` | `Project` | 创建本地项目 |
| `create_remote_project` | `name, ssh_config, source_root` | `Project` | 创建远程项目 |

## 5. UI 设计要点

### 5.1 项目创建向导

- 分步骤向导（Step Wizard）：
  - **本地项目**：基本信息 → 源码路径 → 编译数据库（可选） → 确认
  - **远程项目**：基本信息 → SSH 连接配置 → 连接测试 → 远程源码路径 → Agent 部署 → 确认
- 每步有输入校验和即时反馈
- 支持"返回上一步"修改

### 5.2 项目列表

- 卡片式或列表式布局，可切换
- 每个项目显示：名称、类型图标、源码路径、最后解析状态
- 远程项目额外显示：SSH 连接状态指示灯、服务器地址
- 右键菜单：打开、设置、连接/断开（远程）、删除
- 支持拖拽排序

### 5.3 远程连接面板

- 状态栏区域显示当前 SSH 连接状态
- 连接详情面板：服务器信息、Agent 状态、资源使用率
- 快捷操作：重新连接、Agent 管理、远程终端

### 5.4 远程文件浏览器

- 与本地文件浏览器相同的 UI 风格
- 目录树按需加载（折叠/展开）
- 文件图标区分源文件、头文件、配置文件等
- 加载中状态显示骨架屏或加载动画
- 搜索框支持文件名过滤
