# 01 — 项目管理模块细化需求

> 对应需求规范：§3.1 项目管理（FR1.1 ~ FR1.7）  
> 优先级范围：P0 ~ P1

---

## 1. 模块概述

项目管理模块是用户使用平台的入口，负责本地/远程项目的创建、配置、持久化存储和远程同步。它为后续的源码解析、查询与可视化分析提供基础数据源。

### 1.1 职责边界

- 管理项目的生命周期（创建 → 配置 → 使用 → 归档/删除）
- 管理本地和远程源码的获取与同步
- 管理编译数据库的导入与验证
- 管理认证信息的安全存储
- 提供项目列表与快速切换功能

### 1.2 与其他模块的关系

| 下游模块 | 依赖内容 |
|----------|----------|
| 源码解析与索引 | 项目源码路径、编译数据库路径 |
| 查询与检索 | 项目上下文（当前活跃项目） |
| 可视化分析 | 项目范围、模块定义 |
| AI 智能助手 | 当前项目上下文 |

## 2. 用例详细描述

### UC1.1 创建本地项目

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **前置条件** | 用户已打开应用 |
| **触发方式** | 菜单「文件 → 新建项目 → 本地项目」或首页「新建项目」按钮 |
| **优先级** | P0 |

**主流程**：

1. 用户点击"新建本地项目"。
2. 系统弹出项目创建向导对话框。
3. 用户输入项目名称（必填，不超过 128 字符，不允许特殊字符 `/\:*?"<>|`）。
4. 用户选择本地源码根目录（通过系统文件选择器）。
5. 系统验证目录是否存在且可读。
6. 用户可选导入编译数据库（见 UC1.3）。
7. 用户点击"完成"，系统创建项目配置并保存。
8. 系统自动切换到新创建的项目，在主界面显示项目信息。

**异常流程**：

- **E1**：目录不存在或不可读 → 提示"所选目录无效，请重新选择"。
- **E2**：项目名称已存在 → 提示"项目名称已存在，请使用其他名称"。
- **E3**：目录为空或不含 C/C++ 源文件 → 警告"目录中未检测到 C/C++ 源文件，是否继续？"

**验收标准**：

- [ ] 能成功创建本地项目并保存配置
- [ ] 项目名称校验规则正确生效
- [ ] 目录不存在时给出明确错误提示
- [ ] 项目创建后自动切换为当前活跃项目

### UC1.2 创建远程项目

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色（尤其是远程团队协作成员） |
| **前置条件** | 用户已打开应用，且有可访问的远程仓库 |
| **触发方式** | 菜单「文件 → 新建项目 → 远程项目」 |
| **优先级** | P0 |

**主流程**：

1. 用户点击"新建远程项目"。
2. 系统弹出远程项目创建向导。
3. 用户输入项目名称（规则同 UC1.1）。
4. 用户输入远程仓库地址（URL）。
5. 系统自动检测协议类型（Git/HTTP/HTTPS/SSH/FTP）并显示。
6. 根据协议类型，系统显示对应认证方式：
   - **HTTPS**：用户名 + 密码 / 个人访问令牌（PAT）
   - **SSH**：SSH 密钥文件路径 + 密码短语（可选）
   - **Git**：同 HTTPS 或 SSH
7. 用户填写认证信息。
8. 用户选择本地工作区路径（克隆目标目录），系统可提供默认路径。
9. 用户可选配置分支名（默认：默认分支）、克隆深度（浅克隆 / 全量）。
10. 用户点击"连接测试"，系统尝试连接远程仓库验证认证信息。
11. 连接成功后，用户点击"开始克隆"。
12. 系统后台启动克隆操作，显示进度对话框（见 UC1.7）。
13. 克隆完成后，系统自动创建项目配置并切换到新项目。

**异常流程**：

- **E1**：URL 格式错误 → 提示"请输入有效的仓库地址"。
- **E2**：连接测试失败 → 提示具体错误（认证失败 / 网络不可达 / 仓库不存在）。
- **E3**：磁盘空间不足 → 提示"磁盘空间不足，需要约 X GB 可用空间"。
- **E4**：克隆中断（网络断开） → 保留已下载部分，提示"克隆中断，可稍后重试"。

**验收标准**：

- [ ] 支持 Git、HTTPS、SSH 三种主要协议
- [ ] 认证信息加密存储（不出现在日志和配置明文中）
- [ ] 连接测试能正确反馈成功/失败及原因
- [ ] 克隆进度实时反馈（已接收对象数 / 百分比）
- [ ] 克隆中断可重试，无需从头开始

### UC1.3 导入编译数据库

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **前置条件** | 项目已创建 |
| **触发方式** | 项目创建向导中的可选步骤 / 项目设置面板 |
| **优先级** | P0 |

**主流程**：

1. 用户在项目设置中选择"导入编译数据库"。
2. 系统提供两种方式：
   - **手动选择**：用户通过文件选择器选取 `compile_commands.json` 文件。
   - **自动检测**：系统扫描项目根目录及常见构建目录（`build/`、`out/`、`cmake-build-*/`），自动列出发现的编译数据库文件。
3. 用户选择或确认文件。
4. 系统解析文件，校验 JSON 格式是否合法。
5. 系统统计编译条目数量，显示摘要信息（文件数、编译器路径、主要编译选项）。
6. 系统检查编译数据库中引用的源文件是否存在于项目中。
7. 用户确认导入，系统保存编译数据库路径到项目配置。

**异常流程**：

- **E1**：JSON 格式不合法 → 提示"编译数据库格式错误：<具体解析错误>"。
- **E2**：部分源文件不存在 → 警告"编译数据库中有 N 个文件在项目中未找到，可能影响解析结果"。
- **E3**：文件为空 → 提示"编译数据库为空，请检查文件内容"。

**数据格式要求**：

兼容标准 `compile_commands.json` 格式（JSON 数组），每条记录包含：

```json
{
  "directory": "/path/to/build",
  "command": "gcc -c -o file.o file.c -I/include -DFOO=1",
  "file": "/path/to/source/file.c"
}
```

或使用 `arguments` 数组替代 `command` 字符串。

**验收标准**：

- [ ] 支持 `command` 和 `arguments` 两种格式
- [ ] 自动检测能发现常见构建目录下的编译数据库
- [ ] 格式错误时给出具体的错误位置信息
- [ ] 编译数据库路径持久化保存

### UC1.4 项目配置管理

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **优先级** | P0 |

**配置项清单**：

| 配置项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `name` | String | 是 | 项目名称 |
| `type` | Enum | 是 | `local` / `remote` |
| `source_root` | Path | 是 | 本地源码根目录 |
| `remote_url` | URL | 远程必填 | 远程仓库地址 |
| `remote_protocol` | Enum | 远程必填 | `git` / `https` / `ssh` / `ftp` |
| `auth_info` | Object | 远程必填 | 认证信息（加密存储） |
| `branch` | String | 否 | 分支名，默认 `main` 或仓库默认 |
| `compile_db_path` | Path | 否 | 编译数据库路径 |
| `module_definitions` | Array | 否 | 用户自定义模块定义 |
| `auto_sync` | Boolean | 否 | 是否开启自动同步，默认 `false` |
| `sync_interval` | Integer | 否 | 自动同步间隔（分钟），默认 60 |
| `created_at` | DateTime | 自动 | 创建时间 |
| `updated_at` | DateTime | 自动 | 最后更新时间 |

**持久化方案**：

- 项目配置以 JSON 文件存储在应用数据目录下。
- 路径：`<app_data>/projects/<project_id>/project.json`
- 认证信息单独存储并加密：`<app_data>/projects/<project_id>/auth.enc`
- 加密方式：使用操作系统密钥链（macOS Keychain / Windows Credential Manager / Linux Secret Service）或 AES-256 加密。

**验收标准**：

- [ ] 所有配置项可通过 UI 设置面板修改
- [ ] 配置修改后立即持久化
- [ ] 应用重启后配置完整恢复
- [ ] 认证信息不以明文形式出现在任何日志或配置文件中

### UC1.5 多项目切换

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **优先级** | P1 |

**主流程**：

1. 用户点击左上角的项目选择器（下拉菜单）或侧边栏的项目列表。
2. 系统显示所有已创建的项目列表，包括项目名称、类型图标（本地/远程）、最后更新时间。
3. 用户选择目标项目。
4. 系统切换当前上下文到目标项目：
   - 加载项目配置
   - 加载已有索引（如有）
   - 更新文件浏览器视图
   - 更新图视图（清空或加载上次状态）
5. 如果目标项目尚未解析，系统提示"该项目尚未解析，是否立即开始？"

**验收标准**：

- [ ] 项目切换响应时间 < 2 秒（不含索引加载）
- [ ] 切换时保留原项目的视图状态（下次切回可恢复）
- [ ] 项目列表支持排序（名称/时间）和搜索过滤

### UC1.6 远程同步

| 项目 | 内容 |
|------|------|
| **参与者** | 远程团队协作成员、普通开发者 |
| **前置条件** | 远程项目已创建且首次克隆完成 |
| **优先级** | P0 |

**主流程 — 手动同步**：

1. 用户在远程项目视图中点击"同步"按钮。
2. 系统弹出同步选项：
   - **增量同步**（默认）：仅拉取变更文件（`git pull` / 增量下载）
   - **全量同步**：重新克隆整个仓库
3. 用户选择同步方式并点击"开始同步"。
4. 系统执行同步操作，实时显示进度（见 UC1.7）。
5. 同步完成后，系统显示变更摘要（新增 / 修改 / 删除的文件列表）。
6. 系统自动触发增量解析（仅解析变更文件），更新索引。

**主流程 — 自动同步**：

1. 用户在项目设置中开启"自动同步"并设置间隔。
2. 系统按间隔定时检查远程仓库是否有更新。
3. 若有更新，系统自动执行增量同步。
4. 同步完成后在系统通知区域显示简要结果。

**异常流程**：

- **E1**：认证过期 → 提示"认证信息已失效，请更新"，并引导用户更新认证。
- **E2**：网络中断 → 保存同步状态，恢复后可从断点续传（Git 协议自带支持）。
- **E3**：合并冲突（Git） → 显示冲突文件列表，提供"使用远程版本"/"保留本地"选项，或让用户手动解决。
- **E4**：仓库已删除/移动 → 提示"远程仓库不可访问：<错误详情>"。

**验收标准**：

- [ ] 增量同步仅传输变更文件，耗时远小于全量同步
- [ ] 同步完成后自动触发增量解析
- [ ] 自动同步在后台运行，不阻塞 UI
- [ ] 网络中断后恢复时能继续同步

### UC1.7 同步进度与状态反馈

| 项目 | 内容 |
|------|------|
| **参与者** | 远程团队协作成员 |
| **优先级** | P1 |

**进度信息结构**：

```
SyncProgress {
  phase: "connecting" | "authenticating" | "fetching" | "merging" | "complete" | "error"
  total_objects: u64        // 总对象数（Git 协议）
  received_objects: u64     // 已接收对象数
  total_bytes: u64          // 总字节数
  received_bytes: u64       // 已接收字节数
  percentage: f32           // 进度百分比 0.0 ~ 100.0
  current_file: String      // 当前正在处理的文件名
  speed: String             // 传输速度（如 "2.3 MB/s"）
  eta: String               // 预计剩余时间（如 "约 3 分钟"）
  message: String           // 状态文字描述
}
```

**UI 呈现**：

- 进度条组件：显示百分比和当前阶段文字
- 速度和 ETA 显示在进度条下方
- 完成后显示变更文件摘要
- 错误时显示错误信息和"重试"按钮

**验收标准**：

- [ ] 进度信息刷新频率 ≥ 1 次/秒（有新数据时）
- [ ] 各阶段（连接、认证、拉取、合并）状态清晰可区分
- [ ] 可随时取消同步操作

## 3. 数据模型

### 3.1 项目实体

```
Project {
  id: UUID                      // 唯一标识
  name: String                  // 项目名称
  project_type: ProjectType     // Local | Remote
  source_root: PathBuf          // 本地源码根目录
  remote_config: Option<RemoteConfig>  // 远程配置（仅远程项目）
  compile_db_path: Option<PathBuf>     // 编译数据库路径
  module_definitions: Vec<ModuleDef>   // 模块定义列表
  created_at: DateTime
  updated_at: DateTime
  last_sync_at: Option<DateTime>       // 最后同步时间
  last_parse_at: Option<DateTime>      // 最后解析时间
  parse_status: ParseStatus            // NotStarted | InProgress | Completed | Failed
}
```

### 3.2 远程配置

```
RemoteConfig {
  url: String                   // 远程仓库地址
  protocol: Protocol            // Git | Https | Ssh | Ftp
  branch: String                // 分支名
  clone_depth: Option<u32>      // 克隆深度（None = 全量）
  auth_ref: String              // 指向加密认证信息的引用 ID
  auto_sync: bool               // 是否自动同步
  sync_interval_minutes: u32    // 同步间隔
  local_workspace: PathBuf      // 本地工作区路径
}
```

### 3.3 模块定义

```
ModuleDef {
  name: String                  // 模块名称
  match_type: MatchType         // Directory | FileList | Regex
  patterns: Vec<String>         // 匹配模式列表
  color: Option<String>         // 可视化时的显示颜色
  description: Option<String>   // 模块描述
}
```

## 4. 接口设计（前后端通信）

项目管理模块通过 Tauri Command 暴露以下接口：

| 命令名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| `create_local_project` | `name, source_root` | `Project` | 创建本地项目 |
| `create_remote_project` | `name, url, auth, options` | `Project` | 创建远程项目 |
| `list_projects` | — | `Vec<Project>` | 获取项目列表 |
| `get_project` | `project_id` | `Project` | 获取单个项目详情 |
| `update_project` | `project_id, updates` | `Project` | 更新项目配置 |
| `delete_project` | `project_id` | `bool` | 删除项目 |
| `switch_project` | `project_id` | `Project` | 切换当前活跃项目 |
| `import_compile_db` | `project_id, path` | `CompileDbSummary` | 导入编译数据库 |
| `detect_compile_db` | `project_id` | `Vec<PathBuf>` | 自动检测编译数据库 |
| `test_remote_connection` | `url, auth` | `ConnectionResult` | 测试远程连接 |
| `start_sync` | `project_id, mode` | — | 开始同步（流式返回进度） |
| `cancel_sync` | `project_id` | `bool` | 取消同步 |
| `get_sync_status` | `project_id` | `SyncStatus` | 获取同步状态 |

**事件通道（Tauri Event）**：

| 事件名 | 载荷 | 说明 |
|--------|------|------|
| `sync-progress` | `SyncProgress` | 同步进度更新 |
| `sync-complete` | `SyncResult` | 同步完成通知 |
| `sync-error` | `ErrorInfo` | 同步错误通知 |

## 5. UI 设计要点

### 5.1 项目创建向导

- 分步骤向导（Step Wizard）：基本信息 → 源码路径/远程配置 → 编译数据库（可选） → 确认
- 每步有输入校验和即时反馈
- 支持"返回上一步"修改

### 5.2 项目列表

- 卡片式或列表式布局，可切换
- 每个项目显示：名称、类型图标、源码路径、最后解析状态、最后同步时间
- 右键菜单：打开、设置、同步、删除
- 支持拖拽排序

### 5.3 同步进度面板

- 浮动或嵌入式进度面板
- 可最小化到状态栏
- 同步历史记录可查看

## 6. 错误码定义

| 错误码 | 说明 |
|--------|------|
| `E_PROJ_NAME_INVALID` | 项目名称格式不合法 |
| `E_PROJ_NAME_DUPLICATE` | 项目名称已存在 |
| `E_PROJ_DIR_NOT_FOUND` | 指定目录不存在 |
| `E_PROJ_DIR_NO_PERMISSION` | 目录无读取权限 |
| `E_REMOTE_URL_INVALID` | 远程 URL 格式错误 |
| `E_REMOTE_AUTH_FAILED` | 远程认证失败 |
| `E_REMOTE_UNREACHABLE` | 远程仓库不可达 |
| `E_REMOTE_REPO_NOT_FOUND` | 远程仓库不存在 |
| `E_COMPILE_DB_INVALID` | 编译数据库格式错误 |
| `E_COMPILE_DB_EMPTY` | 编译数据库为空 |
| `E_SYNC_NETWORK_ERROR` | 同步时网络错误 |
| `E_SYNC_CONFLICT` | 同步时发生冲突 |
| `E_SYNC_CANCELLED` | 同步被用户取消 |
| `E_DISK_SPACE_LOW` | 磁盘空间不足 |
