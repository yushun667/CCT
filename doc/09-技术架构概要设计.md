# 09 — 技术架构概要设计

> 本文档基于需求规范 v1.2 和各模块细化需求，概述系统整体技术架构。  
> 详细设计将在开发阶段进一步细化。

---

## 1. 架构总览

### 1.1 技术栈

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| **框架** | Tauri 2.x | Rust 后端 + Web 前端的桌面应用框架 |
| **后端语言** | Rust | 高性能、内存安全、并发友好 |
| **前端框架** | Vue 3 + TypeScript | 组合式 API，强类型，生态丰富 |
| **前端构建** | Vite | 快速开发和构建 |
| **UI 组件库** | 待定（如 Naive UI / Element Plus） | 提供基础 UI 组件 |
| **代码编辑器** | Monaco Editor | VS Code 同源，C/C++ 语法支持 |
| **图渲染** | PixiJS + dagre | WebGL 2D 渲染 + 有向图布局 |
| **解析引擎** | Clang LibTooling (C++ API) | C/C++ 完整语义分析，通过 cxx crate 桥接 Rust |
| **存储** | SQLite (via rusqlite) | 嵌入式关系数据库 |
| **日志** | tracing + tracing-subscriber | 结构化日志框架 |
| **SSH 通信** | russh / ssh2 | SSH 连接、SFTP、远程命令执行 |
| **AI 集成** | async-openai / 自定义 HTTP | LLM API 调用 |
| **MCP** | 自研或社区 SDK | Model Context Protocol 实现 |
| **国际化** | vue-i18n | 前端国际化 |
| **状态管理** | Pinia | Vue 3 状态管理 |

## 3. 关键设计决策

### 3.1 前后端通信

**选择 Tauri IPC 而非 REST API**：

- Tauri IPC 是进程内通信，零网络开销
- 支持 Rust 类型自动序列化（serde）
- 原生支持事件和流式传输
- 类型安全（编译时检查）

### 3.2 解析引擎

**选择 Clang LibTooling (C++ API) 而非 libclang (C API) 或自研解析器**：

- LibTooling 是 Clang 的 C++ 级 AST 访问接口，提供比 libclang (C API) 更完整的语义信息
- 支持 `RecursiveASTVisitor`、`ASTMatcher`、`FrontendAction` 等高级 AST 遍历和匹配能力
- 可直接获取模板实例化、宏展开上下文、完整类型信息等 libclang 难以提供的数据
- 原生支持 `CompilationDatabase`，自动加载 `compile_commands.json` 中的编译选项
- 通过 `cxx` crate 实现 C++/Rust 互操作，C++ 侧编写 AST 遍历逻辑，Rust 侧消费提取结果
- 构建依赖：需要系统安装 LLVM/Clang 开发库，使用 `cmake` 构建 C++ 桥接层

### 3.3 存储引擎

**选择 SQLite 而非自定义格式**：

- 成熟稳定，事务支持好
- SQL 查询表达力强，适合关系数据
- 单文件存储，便于备份和迁移
- 读性能优秀（适合查询密集场景）
- WAL 模式支持读写并发

### 3.4 远程解析

**选择"远程 Agent 就地解析 + 索引回传"而非"拉取代码到本地再解析"**：

- 大型项目（如 Linux Kernel 数十 GB）不适合拉取到本地，占用大量磁盘和带宽
- 远程服务器通常有更强的 CPU 和内存资源，解析速度更快
- 避免本地/远程代码不同步的问题
- 索引数据（SQLite）远小于源码体积，传输高效
- 远程 Agent 与桌面端共享核心解析代码（Rust workspace 共享 crate），保证结果一致

**Agent 通信方式选择 SSH stdin/stdout + JSON-RPC**：

- 不需要 Agent 开放网络端口，安全性高
- 复用 SSH 认证和加密，无需额外的认证/TLS 机制
- 兼容跳板机和防火墙严格的企业网络环境
- JSON-RPC 2.0 协议简单且成熟

### 3.5 图渲染

**选择 WebGL (PixiJS) 而非 SVG / Canvas 2D**：

- WebGL 硬件加速，大规模节点渲染性能好
- PixiJS 提供场景图管理和事件系统
- 支持高 DPI 渲染
- 劣势：文本渲染需要额外处理（Bitmap Font / MSDF）

## 6. 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Clang LibTooling 解析超大项目内存不足 | 解析失败 | 分批解析、内存监控、PCH 缓存 |
| WebGL 图渲染大规模节点卡顿 | 用户体验差 | 聚合策略、LOD、视口裁剪 |
| AI 云端 API 延迟高 | 回答慢 | 支持本地模型、流式输出 |
| 跨平台兼容性问题 | 部分功能失效 | CI 多平台测试、条件编译 |
| 编译数据库缺失 | 解析精度下降 | LibTooling 语法级降级解析、引导用户生成编译数据库 |
| SSH 连接不稳定（网络环境差） | 远程操作中断 | 自动重连、Agent 独立运行、断点续传 |

| 大索引文件传输耗时长 | 用户等待 | zstd 压缩、增量传输、后台传输 |
