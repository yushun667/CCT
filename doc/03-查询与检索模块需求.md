# 03 — 查询与检索模块细化需求

> 对应需求规范：§3.3 查询与检索（FR3.1 ~ FR3.5）  
> 优先级范围：P0 ~ P1

---

## 1. 模块概述

查询与检索模块基于源码解析生成的索引数据，为用户和其他模块提供多维度的代码查询能力。它是连接索引数据与用户交互（可视化、编辑器、AI）的核心桥梁。

### 1.1 职责边界

- 提供符号搜索（名称模糊匹配、类型过滤）
- 提供调用关系查询（调用者/被调用者）
- 提供引用查询（所有引用位置）
- 提供包含关系查询
- 提供路径查询（两点间调用路径）
- 查询结果分页与排序

### 1.2 与其他模块的关系

| 上游模块 | 提供内容 |
|----------|----------|
| 源码解析与索引 | 符号表、关系表、索引 |

| 下游模块 | 消费方式 |
|----------|----------|
| 可视化分析 | 调用图/依赖图的数据源 |
| 代码编辑器 | 符号跳转、悬浮提示 |
| AI 智能助手 | 技能调用（Function Calling） |
| 前端 UI | 搜索结果展示 |

## 2. 用例详细描述

### UC3.1 符号搜索

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **触发方式** | 全局搜索框（快捷键 `Ctrl/Cmd + P`）/ 搜索面板 |
| **优先级** | P0 |

**主流程**：

1. 用户在搜索框中输入查询文本。
2. 系统在用户输入时实时搜索（防抖 200ms）。
3. 系统在符号索引中执行模糊匹配，按相关度排序返回结果。
4. 搜索结果以下拉列表形式展示，每条结果显示：
   - 符号图标（函数/变量/类型/宏）
   - 符号名称（高亮匹配部分）
   - 完全限定名
   - 文件路径 + 行号
   - 符号类型标签
5. 用户可通过过滤器缩小范围（按类型、按文件/目录、按模块）。
6. 用户点击结果项：
   - 跳转到编辑器对应位置
   - 或在图视图中高亮该符号

**搜索算法**：

| 匹配方式 | 说明 | 示例 |
|----------|------|------|
| **前缀匹配** | 以输入开头的符号 | `proc` → `process_request`, `process_data` |
| **子串匹配** | 包含输入的符号 | `req` → `process_request`, `handle_request` |
| **驼峰/蛇形匹配** | 按单词首字母缩写匹配 | `pr` → `process_request`, `ParseResult` |
| **模糊匹配** | 字符顺序匹配，允许间隔 | `prq` → `process_request` |

**相关度排序规则**（权重从高到低）：

1. 完全匹配 → 最高优先
2. 前缀匹配 → 高优先
3. 驼峰/蛇形匹配
4. 子串匹配
5. 模糊匹配 → 最低优先
6. 同等匹配下，定义优先于声明，当前项目优先

**查询接口**：

```
SymbolSearchRequest {
  query: String                  // 搜索文本
  kinds: Option<Vec<SymbolKind>> // 类型过滤
  file_filter: Option<String>    // 文件路径过滤（支持 glob）
  module_filter: Option<String>  // 模块过滤
  definitions_only: bool         // 是否仅搜索定义
  limit: u32                     // 结果数量上限，默认 50
  offset: u32                    // 分页偏移
}

SymbolSearchResult {
  total_count: u64               // 匹配总数
  items: Vec<SymbolSearchItem>   // 结果列表
}

SymbolSearchItem {
  symbol_id: SymbolId
  name: String
  qualified_name: String
  kind: SymbolKind
  file_path: String
  line: u32
  column: u32
  is_definition: bool
  match_score: f32               // 匹配得分 0.0 ~ 1.0
  match_ranges: Vec<(u32, u32)>  // 匹配字符范围（用于高亮）
}
```

**性能要求**：

- 输入响应 < 100ms（返回前 50 条结果）
- 支持百万级符号表的快速检索

**验收标准**：

- [ ] 模糊搜索能正确匹配驼峰命名和蛇形命名
- [ ] 搜索响应时间 < 100ms
- [ ] 搜索结果按相关度正确排序
- [ ] 支持类型、文件、模块过滤
- [ ] 点击结果可跳转到编辑器对应位置

### UC3.2 调用查询

| 项目 | 内容 |
|------|------|
| **参与者** | 架构师、内核/平台开发者、普通开发者 |
| **触发方式** | 右键菜单「查看调用者」/「查看被调用者」/ AI 技能调用 |
| **优先级** | P0 |

**主流程 — 查看调用者（Who calls this?）**：

1. 用户选择一个函数符号（通过搜索/编辑器/图节点）。
2. 系统查询调用关系索引，找到所有 `callee_id == 目标函数` 的记录。
3. 系统返回调用者列表，每条记录包含：
   - 调用者函数名和完全限定名
   - 调用位置（文件、行号）
   - 调用类型（直接 / 虚调用 / 间接）
4. 结果支持按文件分组或按模块分组显示。

**主流程 — 查看被调用者（What does this call?）**：

1. 用户选择一个函数符号。
2. 系统查询 `caller_id == 目标函数` 的记录。
3. 返回被调用者列表，格式同上。

**高级选项**：

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `depth` | 递归查询深度（1 = 直接调用者/被调用者） | 1 |
| `max_results` | 最大返回数量 | 100 |
| `include_indirect` | 是否包含间接调用（函数指针） | true |
| `include_virtual` | 是否包含虚函数调用 | true |
| `group_by` | 分组方式（file / module / none） | none |

**查询接口**：

```
CallQueryRequest {
  function_id: SymbolId          // 目标函数
  direction: CallDirection       // Callers | Callees
  depth: u32                     // 递归深度
  max_results: u32
  include_indirect: bool
  include_virtual: bool
  group_by: GroupBy
}

CallQueryResult {
  target: SymbolInfo             // 目标函数信息
  relations: Vec<CallRelationInfo>
  total_count: u64
}

CallRelationInfo {
  function: SymbolInfo           // 调用者/被调用者
  call_site: Location            // 调用位置
  call_type: CallType            // Direct | Virtual | Indirect
  depth: u32                     // 递归深度层级
}
```

**验收标准**：

- [ ] 能正确查出所有直接调用者和被调用者
- [ ] 递归查询能展示多层调用关系
- [ ] 虚函数调用和间接调用有明确标识
- [ ] 查询响应时间 < 500ms（depth=1）

### UC3.3 引用查询

| 项目 | 内容 |
|------|------|
| **参与者** | 所有用户角色 |
| **触发方式** | 右键菜单「查找所有引用」/ 快捷键 `Shift+F12` |
| **优先级** | P0 |

**主流程**：

1. 用户选择一个符号（函数/变量/类型/宏）。
2. 系统查询引用关系索引。
3. 返回所有引用位置，按文件分组，每条记录包含：
   - 文件路径、行号、列号
   - 引用类型（读 / 写 / 取地址 / 调用 / 类型引用）
   - 所在行的代码片段（上下文预览）
4. 结果面板显示引用总数和分文件统计。

**查询接口**：

```
ReferenceQueryRequest {
  symbol_id: SymbolId
  ref_kinds: Option<Vec<RefKind>>  // 过滤引用类型
  file_filter: Option<String>       // 文件路径过滤
  include_definition: bool          // 是否包含定义位置
  limit: u32
  offset: u32
}

ReferenceQueryResult {
  symbol: SymbolInfo
  total_count: u64
  references: Vec<ReferenceInfo>
}

ReferenceInfo {
  file_path: String
  line: u32
  column: u32
  ref_kind: RefKind
  context_line: String              // 该行代码文本
}
```

**验收标准**：

- [ ] 能找到符号的所有引用位置
- [ ] 引用类型正确标注
- [ ] 支持按引用类型过滤
- [ ] 提供上下文代码预览

### UC3.4 文件包含查询

| 项目 | 内容 |
|------|------|
| **参与者** | 架构师、开发者 |
| **触发方式** | 右键菜单「查看包含关系」/ 文件依赖图入口 |
| **优先级** | P1 |

**查询维度**：

| 维度 | 说明 | SQL 类比 |
|------|------|----------|
| **被包含查询** | "哪些文件 include 了此文件？" | `WHERE target_file = ?` |
| **包含查询** | "此文件 include 了哪些文件？" | `WHERE source_file = ?` |
| **递归包含** | "此文件直接/间接 include 了哪些文件？" | 递归 CTE |

**查询接口**：

```
IncludeQueryRequest {
  file_path: String
  direction: IncludeDirection     // IncludedBy | Includes
  recursive: bool                 // 是否递归
  max_depth: u32                  // 递归深度上限
  exclude_system: bool            // 是否排除系统头文件
}

IncludeQueryResult {
  file: String
  relations: Vec<IncludeInfo>
  total_count: u64
}

IncludeInfo {
  file_path: String
  include_line: u32
  is_system: bool
  depth: u32                      // 递归深度
}
```

**验收标准**：

- [ ] 正确区分直接包含和间接包含
- [ ] 递归查询能展示完整的包含树
- [ ] 支持过滤系统头文件

### UC3.5 路径查询

| 项目 | 内容 |
|------|------|
| **参与者** | 架构师、内核/平台开发者 |
| **触发方式** | 分析菜单「查找调用路径」/ AI 技能调用 |
| **优先级** | P1 |

**主流程**：

1. 用户指定起始函数和目标函数。
2. 系统在调用图中搜索从起点到终点的所有可达路径。
3. 系统返回路径列表（按路径长度排序），每条路径包含沿途的函数序列和调用位置。

**搜索算法**：

- 使用 BFS（广度优先搜索）查找最短路径。
- 使用 DFS + 剪枝 查找所有路径（设置最大深度和最大路径数限制）。
- 对于大型图（> 10 万节点），使用双向 BFS 优化。

**查询接口**：

```
PathQueryRequest {
  source_id: SymbolId             // 起始函数
  target_id: SymbolId             // 目标函数
  max_depth: u32                  // 最大路径深度，默认 20
  max_paths: u32                  // 最大路径数量，默认 10
  algorithm: PathAlgorithm        // ShortestFirst | AllPaths
  exclude_functions: Vec<SymbolId> // 排除的函数（剪枝）
}

PathQueryResult {
  source: SymbolInfo
  target: SymbolInfo
  paths: Vec<CallPath>
  search_stats: PathSearchStats   // 搜索统计（探索节点数、耗时）
}

CallPath {
  steps: Vec<PathStep>
  length: u32
}

PathStep {
  function: SymbolInfo
  call_site: Location             // 到下一步的调用位置
}
```

**验收标准**：

- [ ] 能找到存在的最短调用路径
- [ ] 路径中的每一步都有明确的调用位置信息
- [ ] 大型图的查询时间 < 5 秒
- [ ] 不存在路径时给出明确提示

## 4. 错误码定义

| 错误码 | 说明 |
|--------|------|
| `E_QUERY_NO_INDEX` | 项目尚未建立索引 |
| `E_QUERY_SYMBOL_NOT_FOUND` | 指定符号不存在 |
| `E_QUERY_TIMEOUT` | 查询超时 |
| `E_QUERY_PATH_NOT_FOUND` | 两函数间无可达路径 |
| `E_QUERY_DEPTH_EXCEEDED` | 递归深度超限 |
| `E_QUERY_TOO_MANY_RESULTS` | 结果数量超限 |
