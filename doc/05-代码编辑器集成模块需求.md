# 05 — 代码编辑器集成模块细化需求

> 对应需求规范：§3.5 代码编辑器集成（FR5.1 ~ FR5.3）  
> 优先级范围：P0 ~ P1

---

## 1. 模块概述

代码编辑器集成模块在应用中嵌入功能完善的代码编辑器，提供源码查看、语法高亮、符号导航等能力，并实现与可视化图的双向联动，构成"图 ↔ 代码"的闭环交互体验。

### 1.1 职责边界

- 嵌入代码编辑器组件（基于 Monaco Editor 或 CodeMirror）
- 提供 C/C++ 语法高亮和代码折叠
- 实现从图到代码的跳转（图 → 编辑器）
- 实现从代码到图的联动（编辑器 → 图）
- 提供符号悬浮提示（Hover Tooltip）
- 支持多标签页打开多个文件

### 1.2 与其他模块的关系

| 关联模块 | 交互方式 |
|----------|----------|
| 可视化分析 | 双向：图节点 ↔ 编辑器定位 |
| 查询与检索 | 搜索结果点击 → 编辑器定位 |
| 源码解析与索引 | 提供符号信息用于高亮、提示 |
| AI 智能助手 | 当前查看文件/符号作为 AI 上下文 |

#### 2.1.1 编辑器选型

#### 2.1.3 多标签页管理

```
┌──────────────────────────────────────────────────────┐
│ [main.c ×] [handler.cpp ×] [*server.h ×]  [+]       │
├──────────────────────────────────────────────────────┤
│  src/handler.cpp > MyNamespace > MyClass > handle()  │ ← 面包屑
├──────────────────────────────────────────────────────┤
│  1 │ #include "server.h"                              │
│  2 │                                                  │
│  3 │ namespace MyNamespace {                          │
│  4 │                                                  │
│  5 │ void MyClass::handle(const Request& req) {       │ ← 高亮行
│  6 │     auto result = process(req);                  │
│  7 │     send_response(result);                       │
│  8 │ }                                                │
│    │                                                  │
└──────────────────────────────────────────────────────┘
```

- 标签页支持拖拽重排
- 右键标签页：关闭、关闭其他、关闭所有、复制路径
- 未保存标签显示修改标记（当前场景为只读，该项保留扩展）
- 标签页超出宽度时支持滚动或折叠菜单

### UC5.2 双向跳转

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |

#### 2.2.1 图 → 编辑器（跳转到源码）

**触发方式**：

| 操作 | 效果 |
|------|------|
| 双击图节点 | 打开对应文件，定位到符号定义位置 |
| 右键 → "跳转到源码" | 同上 |
| 搜索结果点击 | 打开对应文件，定位到匹配位置 |
| 引用列表项点击 | 打开对应文件，定位到引用位置 |

**定位行为**：

1. 如果文件已打开，切换到对应标签页。
2. 如果文件未打开，在新标签页中打开。
3. 滚动到目标行，使目标行位于视口垂直居中位置。
4. 高亮目标行（黄色背景闪烁 2 秒后淡出）。
5. 如果是函数定义，将光标置于函数名处。

**跳转接口**：

```
EditorNavigateRequest {
  file_path: String              // 文件路径
  line: u32                      // 目标行
  column: u32                    // 目标列
  highlight: bool                // 是否高亮目标行
  preview: bool                  // 是否为预览模式（预览模式下不新建标签，复用预览标签）
}
```

#### 2.2.2 编辑器 → 图（显示调用关系）

**触发方式**：

| 操作 | 效果 |
|------|------|
| 光标置于函数名上 + 右键 → "显示调用图" | 在图视图中以该函数为中心展示调用图 |
| 光标置于函数名上 + 右键 → "显示调用者" | 在结果面板中显示调用者列表 |
| 光标置于函数名上 + 右键 → "显示被调用者" | 在结果面板中显示被调用者列表 |
| 快捷键 `Ctrl+Shift+G` | 在图视图中定位当前函数 |

**符号识别流程**：

1. 获取光标位置（文件、行号、列号）。
2. 在索引中查询该位置的符号：
   - 精确匹配：查找包含该位置的符号定义/引用。
   - 如果光标位于标识符上，提取标识符文本并在索引中搜索。
3. 若找到唯一符号，直接触发对应操作。
4. 若找到多个候选符号（如重载函数），弹出选择菜单。
5. 若未找到符号，提示"当前位置未识别到已索引的符号"。

**编辑器右键菜单扩展**：

```
编辑器右键菜单：
├── 复制
├── 全选
├── ─────────────
├── 跳转到定义          (Ctrl+Click / F12)
├── 查找所有引用        (Shift+F12)
├── ─────────────
├── 显示调用图          (Ctrl+Shift+G)
├── 显示调用者
├── 显示被调用者
├── 查找调用路径...
├── ─────────────
├── 在终端中打开文件位置
└── 复制文件路径
```

**验收标准**：

- [ ] 图节点双击可正确跳转到编辑器对应位置
- [ ] 跳转时文件自动打开（若未打开）并滚动到目标行
- [ ] 编辑器中可通过右键菜单触发图视图操作
- [ ] 符号识别准确，多义符号时提供选择
- [ ] 跳转响应时间 < 500ms

### UC5.3 符号信息悬浮提示

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |

**触发方式**：鼠标悬停在标识符上 500ms 后显示提示框。

**提示内容**：

```
┌────────────────────────────────────────┐
│ int process_request(const Request& req)│  ← 函数签名
│                                        │
│ 定义: src/handler.cpp:42               │  ← 定义位置
│ 所属: MyNamespace::MyClass             │  ← 所属类/命名空间
│ 调用者: 5 | 被调用: 12 | 引用: 8       │  ← 统计信息
│                                        │
│ [跳转定义] [查看调用者] [查看引用]       │  ← 快速操作链接
└────────────────────────────────────────┘
```

**不同符号类型的提示内容**：

| 符号类型 | 显示内容 |
|----------|----------|
| **函数** | 签名、定义位置、调用者数、被调用数、复杂度 |
| **变量** | 类型、定义位置、作用域、引用数 |
| **类型（struct/class）** | 类型定义、成员数、基类、大小 |
| **宏** | 宏体、定义位置、引用数 |
| **枚举** | 枚举值列表、定义位置 |

**实现方案**：

1. 编辑器注册 Hover Provider。
2. 当鼠标悬停时，提取光标位置的标识符。
3. 通过 Tauri Command 查询索引，获取符号信息。
4. 组装 Markdown 格式的提示内容并返回给编辑器。

**性能要求**：

- 悬浮提示查询响应 < 200ms
- 提示框出现/消失有平滑动画
- 鼠标移开后提示框延迟 300ms 消失（移入提示框内则保持显示）

**验收标准**：

- [ ] 悬停函数名显示完整签名和统计信息
- [ ] 悬停变量名显示类型和作用域
- [ ] 悬停类型名显示类型定义摘要
- [ ] 提示框中的快速操作链接可正常触发
- [ ] 悬浮提示响应 < 200ms

## 3. 编辑器配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `editor.fontSize` | Number | 14 | 字体大小 |
| `editor.fontFamily` | String | `"Consolas, 'Courier New', monospace"` | 字体族 |
| `editor.tabSize` | Number | 4 | Tab 宽度 |
| `editor.wordWrap` | Enum | `off` | 自动换行（off / on / bounded） |
| `editor.minimap` | Boolean | true | 是否显示小地图 |
| `editor.lineNumbers` | Enum | `on` | 行号显示（on / off / relative） |
| `editor.theme` | Enum | `vs-dark` | 编辑器主题（亮色/暗色/自定义） |
| `editor.renderWhitespace` | Enum | `selection` | 空白字符显示 |

## 4. 接口设计

| 命令名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| `read_file` | `file_path` | `FileContent` | 读取文件内容 |
| `get_symbol_at_position` | `file_path, line, column` | `Option<SymbolInfo>` | 获取指定位置的符号信息 |
| `get_hover_info` | `file_path, line, column` | `HoverInfo` | 获取悬浮提示信息 |
| `get_breadcrumb` | `file_path, line` | `Vec<BreadcrumbItem>` | 获取面包屑导航数据 |

**前端事件**：

| 事件名 | 载荷 | 说明 |
|--------|------|------|
| `editor-navigate` | `EditorNavigateRequest` | 请求编辑器跳转 |
| `editor-symbol-selected` | `SymbolId` | 用户在编辑器中选择了符号 |
| `editor-file-opened` | `file_path` | 文件被打开 |
| `editor-cursor-changed` | `file_path, line, column` | 光标位置变化 |

## 5. 错误码定义

| 错误码 | 说明 |
|--------|------|
| `E_EDITOR_FILE_NOT_FOUND` | 文件不存在 |
| `E_EDITOR_FILE_TOO_LARGE` | 文件过大（> 10MB），建议分段加载 |
| `E_EDITOR_ENCODING` | 文件编码不支持 |
| `E_EDITOR_SYMBOL_NOT_FOUND` | 指定位置未找到符号 |
